LYNX is a fully functional ride-management platform written in C++ that demonstrates how fundamental data structures power real-world software. Every structure—linked lists, queues, stacks, hash tables, AVL trees, heaps, and graphs—is implemented from scratch without the Standard Template Library. The system manages the complete lifecycle of rides, from passenger registration and driver assignment through route calculation and trip history.

## What LYNX Demonstrates

LYNX maps each data structure to a concrete business problem, making it an ideal reference for understanding how theory translates to practice.

<CardGroup cols={3}>
  <Card title="Queue → Waiting List" icon="clock" href="/data-structures/queue">
    Pending trips are managed as a FIFO queue—first requested, first assigned.
  </Card>
  <Card title="DoublyLinkedList → Active Trips" icon="arrows-left-right" href="/data-structures/doubly-linked-list">
    In-progress rides live in a doubly-linked list for fast insertion and removal at any position.
  </Card>
  <Card title="Stack → Trip History" icon="clock-rotate-left" href="/data-structures/stack">
    Completed and cancelled trips are pushed onto a LIFO stack for instant last-trip lookup.
  </Card>
  <Card title="Hash Table → Fast Lookups" icon="bolt" href="/data-structures/hash-table">
    Passengers and drivers are indexed by DNI for O(1) average-case authentication.
  </Card>
  <Card title="AVL Tree → Range Queries" icon="tree" href="/data-structures/avl-tree">
    Trip price and driver rating range searches run in O(log n) via a self-balancing AVL tree.
  </Card>
  <Card title="Graph → City Routing" icon="map-location-dot" href="/data-structures/graph">
    Cities and road connections form a weighted graph; Dijkstra finds the shortest route.
  </Card>
</CardGroup>
